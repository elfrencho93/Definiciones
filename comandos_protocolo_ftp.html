5 Comandos del Protocolo FTP


Comando Descripción Ejemplo de Uso
open Establece una conexión con el servidor FTP en el host especificado por el parámetro HostName. Si se especifica el número de puerto opcional, el comando ftp intenta conectarse a un servidor en ese puerto. Si la función de inicio de sesión automático está configurada (es decir, el indicador -n no se especificó en la línea de comandos), el comando ftp intenta iniciar la sesión del usuario en el servidor FTP. Sintaxis:
open <computer> [<port>]

<computer>: Especifica el ordenador remoto a el que está intentando conectarse. Puede usar una dirección IP o un nombre de ordenador (en cuyo caso debe estar disponible un servidor DNS o un archivo Hosts).

[<port>]: Especifica un número de puerto TCP que se usará para conectarse a un servidor ftp. De forma predeterminada, se utiliza el puerto TCP 21.

Para conectarse al servidor ftp en ftp.microsoft.com, escriba:
open ftp.microsoft.com

Para conectarse al servidor ftp en ftp.microsoft.com que está escuchando en el puerto TCP 755, escriba:
open ftp.microsoft.com 755
append Agrega un archivo local a un archivo en el host remoto. Si no se especifica el nombre del archivo remoto, se usa el nombre del archivo local, modificado por cualquier configuración realizada con el subcomando ntrans o el subcomando nmap. El subcomando append usa los valores actuales para los subcomandos form, mode, struct y type al agregar el archivo. Sintaxis:
append <localfile> [<remotefile>]

<localfile>: Especifica el archivo local para agregar.

[<remotefile>]: Especifica el archivo en el equipo remoto al que se agrega <localfile>. Si no usa este parámetro, el nombre del <localfile> se usa en lugar del nombre del archivo remoto.

Para agregar file1.txt a file2.txt en la computadora remota, escriba:
append file1.txt file2.txt

Para agregar el archivo1.txt local a un archivo llamado archivo1.txt en el ordenador remoto.
append file1.txt
hash Alterna la impresión del signo de almohadilla (#). Cuando el subcomando hash está activado, el comando ftp muestra un signo hash por cada bloque de datos (1024 bytes) transferido. Sintaxis:
hash

Para alternar la impresión del signo de número (#) para cada bloque de datos que se transfiere, escriba:
hash
verbose Alterna el modo detallado. Cuando el modo detallado está activado (predeterminado), el comando ftp muestra todas las respuestas del servidor FTP remoto. Además, el comando ftp muestra estadísticas sobre todas las transferencias de archivos cuando se completan. Sintaxis:
verbose

Para activar y desactivar el modo detallado, escriba:
verbose
user Identifica al usuario local (Usuario) en el servidor FTP remoto. Si no se especifica el parámetro Contraseña o Cuenta y el servidor remoto lo requiere, el comando ftp solicita la contraseña o la cuenta localmente. Si se requiere el parámetro Cuenta, el comando ftp lo envía al servidor remoto después de que se completa el proceso de inicio de sesión remoto. Sintaxis:
user <usuario> [<contraseña>] [<cuenta>]

<usuario>: Especifica un nombre de usuario con el que iniciar sesión en el equipo remoto.

[<contraseña>]: Especifica la contraseña para el nombre de usuario. Si no se especifica una contraseña pero es necesaria, el comando ftp solicita la contraseña.

[<cuenta>]: Especifica una cuenta con la que iniciar sesión en el equipo remoto. Si no se especifica una cuenta pero se requiere, el comando ftp solicita la cuenta.

Para especificar Usuario1 con la contraseña Contraseña1, escriba:

user Usuario1 Contraseña1
